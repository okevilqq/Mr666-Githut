## Project Identity

**链商2.0 · 链生活品牌** — 面向社区商业的数字经营平台（Community Business Digital Operating Platform），通过商户独立经营、生态会员互通、消费权益流转和真实交易激励机制，帮助商家提升复购率、帮助用户获得持续消费权益、帮助社区形成可持续商业循环。

**两大体系：** ① 消费流转体系：平台商家 → 联盟商家 → 综合商城（软性优先级引导）② 全生态会员权益互通体系：代金券/积分/消费金全平台通用

The brand "链生活" (Chain Life) operates under 链邦科技, a subsidiary of 全球拼购 (GGbingo).

**Role:** 企业宣传策划专员 (Corporate Communications Planning Specialist) — 梁君衡

## Key Project Conventions

### Document generation patterns:
- All generation scripts follow the same pattern: `const docx = require('docx')` → define helper functions (`h1()`, `h2()`, `p()`, `b()`, `dataTable()`, `divider()`) → build Document → `Packer.toBuffer()` → `fs.writeFileSync()`
- Color constants defined at top: `C.MAIN='#1A5276'`, `C.HEADER='#1a1a2e'`, etc.
- All documents output to `20260602 链商平台 技术部会议整理/` directory
- Font: `微软雅黑`, size 21pt body text

### Brand color system:
- 主色 (Primary): `#1A5276` (深海蓝)
- 强调色 (Accent): `#E67E22` (温暖橙)
- 合规色 (Compliance): `#C0392B` (red for redlines), `#1E8449` (green for compliant)
- 辅助: `#2C3E50` (dark), `#EBF5FB` (light blue bg), `#7F8C8D` (gray)

### Compliance terminology (MANDATORY):
All brand/business documents MUST use compliant terminology per the legal framework:
- "数字资产" → "消费权益" or "会员权益"
- "数字信用债券" → "商家营销额度"
- "资产增值" → "权益升级"
- "消费信用分" → "消费活跃度指数"
- "分润算法优化引擎" → "商家服务费计算规则"
- "收益/盈利" → "服务费/增收"（不可暗示经济回报）
- "红利/增值回报" → "权益升级/消费权益"
- "资本(化)" → "资源/产业"（不可暗示金融操作）
- "变现" → "转化"
- "创业" → "参与/开展推广"（不可暗示零门槛创业）
- "全球化" → "全国化/本地化"（当前阶段聚焦国内）
- "数据确权" → "数据权益管理"
- "数据即资产" → "数据驱动经营"
- NEVER use: "币", "Token", "通证", "投资回报", "稳赚", "躺赚", "数字资产交易", "资本集群", "资本化运作"

### Three compliance red lines (from 6/2 tech meeting):
1. 积分不可兑现 (points cannot be converted to cash)
2. 不可形成资金池 (no capital pool without payment license)
3. 不可承诺收益 (no guaranteed returns in any language)

### V3.2 Key Model Parameters (for document cross-reference) — unchanged from V3.1:
- **Payment channel costs**: 汇付收单 0.6%, 余额支付 0.1%, 消费金核销 0%
- **Secondary withdrawal cost**: 0.1% (微信商户号→银行账户), applies to 商家/平台/服务站/推广者/城市服务商
- **Platform base distribution** (汇付收单): 渠道0.6% + 商家75.4% + 平台5% + 服务站+推广者9% + 城市服务商1% + 消费金3% + 营销池4% + 风控备用金2% = 100%
- **Alliance base distribution** (汇付收单): 渠道0.6% + 商家71.4% + 平台渠道费4.5% + 平台5% + 服务站+推广者10% + 城市服务商1% + 消费金2% + 营销池3.5% + 风控备用金2% = 100%
- **E-commerce base distribution** (汇付收单): 渠道0.6% + 商家77.9% + 平台6% + 服务站+推广者6% + 城市服务商1% + 消费金3% + 营销池3.5% + 风控备用金2% = 100%
- **3-tier management**: 城市服务商(区域合伙人) → 服务站(社区节点) → 推广者(一线推广), 服务站佣金池内部分配35:65

- **Breakeven**: 40,500笔/月, Net margin: 0.89%
- **🆕 Cross-store universal benefits**: 代金券/积分/消费金全平台通用——营销池统一结算, 跨店通兑不构成二清(汇付直清)

### 链商2.0 定位与两大体系（V3.2 内测反馈更新）:
- **链商2.0定位**: 面向社区商业的数字经营平台——商户独立经营·生态会员互通·消费权益流转·真实交易激励
- **三大目标**: 帮助商家提升复购率 · 帮助用户获得持续消费权益 · 帮助社区形成可持续商业循环
- **消费流转体系**: 平台商家(优先展示入口)→联盟商家(延伸履约网络)→综合商城(补充供给)——软性优先级(+2/0/-2位搜索权重), 非强制路径
- **全生态会员权益互通体系**: 消费者在任一商家获得权益(代金券/积分/消费金)均可在全平台任意商家使用——在A店获得、在B店使用
- **交易即分润**: 平台仅在实际交易发生时获取服务费——零固定费用、零预充值、零资金占用

### 千面千店 页面设计层级（V3.2）:
- **平台商家**: Premium layout, 高信息密度, 搜索加权+2, 品牌认证蓝标——深度自定义(品牌色/故事/视频/相册)
- **联盟商家**: Standard layout, 标准信息密度, 搜索基准位, 社区好店绿标——6套模板+轻定制(品牌色/横幅)
- **综合商城**: Minimal layout, 极简密度, 搜索加权-2, 平台自营灰标——标准化模板, 不可自定义
- **软性优先级**: 消费者可自由浏览所有商家——仅默认排序有优先, 可通过筛选/搜索/分类/地图找到任何商家
