# 服务费结算与核销模型演进史

## V1.0（20260603）— 初始分析版

**来源：** `generate_settlement_analysis.js`

首次系统性审计。覆盖三大业态（平台/联盟/电商）× 四种场景（汇付收单/余额支付/消费金核销/混合支付）= 12个子模型。

**核心发现：**
- 12个问题：4个P0（必须修复）+ 4个P1（建议修复）+ 4个P2（优化建议）
- 初步识别了"二清风险"、"资金池风险"、"税务合规"等关键问题
- 提出了基础的分账架构

## V2.0（20260604）— 会议共识版

**来源：** `generate_revised_settlement_model.js`
**触发：** 6/4 技术部会议

**关键决策（6/4会议共识）：**
1. **无充值** — 消费者不可向平台充值，杜绝资金池
2. **无资金池** — 所有资金由汇付直清，平台不持留任何资金
3. **扣除渠道成本** — 分账前先扣除支付渠道费用（汇付0.6%）
4. **六方利益分配** — 消费者+商家+平台+推广者+服务站+支付渠道，各方利益独立核算

**模型结构：**
- 平台基础分账（汇付收单）：渠道0.6% + 商家75.4% + 平台5% + 服务站+推广者9% + 营销池10%
- 联盟基础分账：略异于平台
- 电商基础分账：略异于平台
- 平台经营指标：利润/毛利率/盈亏平衡点
- 敏感性分析

## V3.0 → V3.1（20260606）— 内测反馈修正

**来源：** `generate_v3_settlement_model.js`
**触发：** 模型后台校验 + 内测反馈

**关键变更：**
- 引入"城市服务商"角色，从服务站佣金池中分配1%
- 风控备用金独立为2%（汇付托管，不可动用）
- 营销池从10%调整为3.5-4%（受渠道成本和备用金挤压）
- 盈亏平衡点重新测算：40,500笔/月
- 净利率：0.89%

## V3.2（20260609）— 跨店通兑版（当前最新）

**来源：** `generate_v3_settlement_model.js`（已更新至V3.2）
**触发：** 链商2.0定位升级——全生态会员权益互通

**🆕 关键变更：**
1. **跨店通用权益结算** — 代金券/积分/消费金全平台通用，营销池统一结算
2. **消费流转体系** — 平台商家→联盟商家→综合商城（软性优先级引导）
3. **千面千店规范** — 三种商家类型的页面设计层级
4. **交易即服务费结算** — 零固定费用、零预充值、零资金占用
5. **跨店通兑不构成二清** — 汇付直清，法律意见明确
6. **100%去金融中心化** — 所有资金流转经持牌支付机构
7. **KPI Scorecard** — 5维34项指标体系

**参数不变处（与V3.1完全一致）：**
- 三种业态分账比例不变
- 三级管理体系不变（城市服务商→服务站→推广者）
- 三元营销参数不变（代金券5%/30%/90天，积分1:100/20%/2年，消费金30%/12月）
- 盈亏平衡40,500笔/月不变
