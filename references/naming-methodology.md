# 品牌命名方法论与评估框架

> 链商平台（赋商通·链生活）品牌命名的方法论基础、评估维度、商标检索策略及命名演进全记录。

**版本：** V1.0 | **编制：** Mr666 | **日期：** 2026-06-15

## 一、命名方法论总览

本项目采用「6大核心维度 × 6种行业命名模型」双轴评估体系。6大维度评估一个名字的"好不好"，6种模型锚定一个名字的"对不对"。

---

## 二、6大核心维度（评估任何候选名称）

### 维度1：核心定位元素 — 锚定品牌底层基因

这是命名的根基，决定名字的内核调性，避免名字与品牌脱节。

| 子维度 | 说明 | 链商项目对应要求 |
|--------|------|-----------------|
| 赛道属性 | 精准匹配行业、品类、业务场景 | 本地生活商业 / 商家运营管理 / SaaS数字化 |
| 品牌调性 | 贴合品牌核心风格 | 专业可信 + 务实温暖 + 科技赋能 |
| 核心价值 | 嵌入品牌卖点、理念、初心 | 帮助商家留客、联盟共营、跨店通兑 |
| 目标人群 | 适配受众审美 | B端中小商户决策者（店主/店长）+ C端社区消费者 |

### 维度2：传播记忆元素 — 降低品牌传播成本

| 子维度 | 说明 | 链商项目对应要求 |
|--------|------|-----------------|
| 简洁性 | 2-4字中文，规避生僻字、多音字 | 候选均控制在3-4字 |
| 顺口度 | 音律平仄搭配协调，无拗口连读、无负面谐音 | 经普通话及主要方言谐音检查 |
| 独特性 | 规避通用词、网红泛滥词，具备差异化辨识度 | 与美团/口碑/有赞等形成明显区隔 |
| 视觉感 | 字形结构美观，适合Logo设计与VI延展 | 候选均评估过字形 |

### 维度3：品牌延展性 — 支持未来业务扩张

| 子维度 | 说明 |
|--------|------|
| 子品牌兼容 | 可衍生 XX·商家版、XX·会员端、XX·城市服务商 等 |
| 品类覆盖 | 名称不局限于单一业态（餐饮/零售/服务均适用） |
| 地域扩展 | 从单一城市→全国→（未来）国际化均适用 |
| 英文名 | 辅助英文名国际化友好、无不良含义 |

### 维度4：商标可注册性 — 法律基础

| 子维度 | 说明 |
|--------|------|
| 显著性 | 非通用词、非描述性词汇——具备商标法要求的"显著特征" |
| 在先权利 | 无他人在核心类别（35/42/9类）的相同或近似注册 |
| 防御空间 | 核心类别可注册 + 关联类别（16/38/41/45类）有防御注册空间 |
| 域名可用 | .com/.cn 域名可注册或可购买 |

### 维度5：行业对标性 — 赛道定位准确

名字需在本地生活/商家服务赛道中找到合理坐标，不与金融/区块链/电商产生错误联想。

### 维度6：情感共鸣 — 用户心理认同

名字需唤起正向情感——信任、温暖、专业、可靠——而非冷漠、浮夸、投机。

## 三、6种本地生活平台命名模型

基于对中国主流本地生活平台（美团、口碑、饿了么、高德、有赞、微盟等）的命名分析，提炼6种模型：

### 模型1：联盟共营型
- **特征：** 强调"联盟""共同""互通"概念——名字传递多商家协作的生态感
- **关键词：** 联、盟、共、合、通、汇、聚
- **代表案例：** 美团（美好团聚）、口碑（口口相传·碑碑相印证）
- **链商适配：** ⭐⭐⭐⭐⭐ 最匹配——平台核心就是商家联盟共营
- **对应方案：** 域营、赋商联、邻汇通

### 模型2：商家经营型
- **特征：** 强调"帮助商家经营"——名字传递SaaS/工具/管理属性
- **关键词：** 店、商、营、智、慧、管、帮
- **代表案例：** 有赞（帮助商家获得赞美）、微盟（微信生态商业联盟）
- **链商适配：** ⭐⭐⭐⭐ 很匹配——B端定位核心
- **对应方案：** 店智汇、店营通

### 模型3：位置互通型
- **特征：** 强调"附近""邻里""社区"——名字传递地理位置的亲近感
- **关键词：** 邻、近、社、区、汇、达、通
- **代表案例：** 高德（高地·德行）、饿了么（到家场景）
- **链商适配：** ⭐⭐⭐ 部分匹配——社区商业有位置属性但非全部
- **对应方案：** 邻汇通、近商联

### 模型4：科技赋能型
- **特征：** 强调"数字化""智能""AI"——名字传递技术平台属性
- **关键词：** 数、智、云、科、技、AI、+
- **代表案例：** 微盟（Weimob——WeChat + Mobile）
- **链商适配：** ⭐⭐ 局部匹配——SaaS是手段但不是品牌全部
- **对应方案：** 店智汇（"智"字的科技属性）

### 模型5：消费场景型
- **特征：** 强调"消费""生活""服务"——名字传递消费者端的价值感知
- **关键词：** 生活、消费、惠、省、便、好
- **代表案例：** 美团、口碑、饿了么
- **链商适配：** ⭐ 不匹配——链商定位是B端经营而非C端消费

### 模型6：文化意境型
- **特征：** 强调"文化""匠心""传承"——名字传递品牌厚度和差异化
- **关键词：** 砚、璞、耕、赋、初、源
- **代表案例：** 豆瓣、知乎、网易
- **链商适配：** ⭐⭐ 差异化为辅——文化字可规避高频饱和字
- **对应方案：** 赋业初、砚璞启、耕赋源

## 四、商标检索策略

### 4.1 核心注册类别（P0 必须）

| 类别 | 内容 | 重要性 |
|------|------|--------|
| **第35类** | 广告、商业管理、商业经营 | 🔴 核心——平台运营必需 |
| **第42类** | 科学技术服务、SaaS、软件即服务 | 🔴 核心——技术平台必需 |
| **第9类** | APP、小程序、可下载软件 | 🔴 核心——产品上架必需 |

### 4.2 防御注册类别（P1 建议）

| 类别 | 内容 |
|------|------|
| 第16类 | 印刷品、宣传册 |
| 第36类 | 金融事务（防御性注册——防止被金融类抢注） |
| 第38类 | 电信、信息传输 |
| 第41类 | 教育、培训、娱乐 |
| 第45类 | 法律服务、社交网络 |

### 4.3 商标检索排查清单（24项）

1. 中文名称精确检索（商标局官网）
2. 中文名称近似检索（同音/形近/偏旁替换）
3. 英文名称精确检索
4. 英文名称近似检索
5. 第35类在先权利核查
6. 第42类在先权利核查
7. 第9类在先权利核查
8. 第36类防御核查
9. 企查查/天眼查公司名冲突检索
10. 百度/微信指数品牌词检索
11. .com 域名可用性
12. .cn 域名可用性
13. 小程序名称可用性
14. 公众号名称可用性
15. 抖音/快手/小红书品牌号可用性
16. 百度百科词条冲突
17. 主要方言负面谐音检查（粤语/闽南语/客家话/四川话）
18. 英文名国际负面含义检查
19. 英文名国际化发音检查
20. 与知名品牌（本地生活/电商/金融类）近似性评估
21. Logo设计可行性评估
22. 第16类防御核查
23. 第38类防御核查
24. 第41/45类防御核查

### 4.4 高频饱和字（应避开）

基于第35/42/9类商标数据库分析，以下汉字在相关类别中注册密度极高，新品牌名应尽量避开：

> 商、通、联、汇、智、云、达、信、诚、鑫、泰、安、优、佳、美、乐、享、宝、金、银

## 七、命名决策原则

1. **B端优先：** 名字传递"帮助商家经营"，非"让消费者购物"
2. **合规避嫌：** 避开金融/区块链/虚拟货币联想
3. **商标先行：** 先查后定——未经商标检索的名字不进入候选
4. **字频分析：** 优先使用低频高语义字，避开饱和字
5. **3字黄金：** 2-4字可接受，3字最佳（兼顾简洁与表达）
6. **方言审查：** 须通过粤语/闽南语/客家话负面谐音检查
7. **域名同步：** 定名前检查 .com/.cn 可用性
