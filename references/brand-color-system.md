## 五、色彩使用禁忌

1. **不可混用合规红与链商红** — `#C0392B`（警示）和 `#D62828`（品牌）语义不同，不可互相替代
2. **正文不可用纯黑** — 始终使用 `#333333`，纯黑 `#000000` 对比度过高，不适于阅读
3. **表格表头不可用彩色文字** — 表头深色底必须配白色文字
4. **不可在浅蓝背景上放置科技蓝文字** — `#EBF5FB` 与 `#1F5EFF` 对比度不足
5. **金色仅用于会员/权益相关元素** — `#D4A843` 不可滥用于普通按钮或标题
6. **千面千店商家自定义色不可使用** — `#C0392B`（合规红）和平台品牌主色 `#1A5276`（避免冒充平台认证）

## 六、工具链色彩引用

在生成脚本中统一通过 `lib/constants.js` 的 `COLORS` 常量引用色彩：

```javascript
const { COLORS } = require('./lib/constants');
// COLORS.DEEP_BLUE   → '#1A5276'
// COLORS.CHAIN_RED    → '#D62828'
// COLORS.TECH_BLUE    → '#1F5EFF'
// COLORS.WARM_ORANGE  → '#E67E22'
// COLORS.RED          → '#C0392B'
// COLORS.GREEN        → '#1E8449'
// COLORS.YELLOW       → '#F39C12'
// COLORS.GOLD         → '#D4A843'
```

旧脚本中的 `C` 对象（来自 `lib/docx-helpers.js`）为兼容层，新脚本应迁移至 `COLORS`。

---

*本文件与 `lib/constants.js` 的 COLORS 部分保持同步。如有色彩增删，需同时更新两处。*
