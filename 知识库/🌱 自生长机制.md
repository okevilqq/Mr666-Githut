# 🌱 知识库自生长机制

## 核心理念

> **每一个项目产出，自动成为知识库的一个节点；每一条思考记录，自动丰富知识图谱。**

知识库不是静态存档——它随着项目推进**自我生长**。

---

## 五条生长路径

### 1. 产出自动索引
```
generate_*.js → .docx/.xlsx 输出
                ↓
regenerate_all.sh → obsidian_auto_log.sh
                ↓
📝 变更日志.md ← 自动记录
                ↓
🏠 首页 Dataview ← 自动显示
```

### 2. 日报累积演化
```
每日写日报（模板）→ 积累 → 周报综述 → 月报总结 → 阶段报告
```

### 3. 知识碎片 → 结构化笔记
```
收件箱（快速捕获）→ 知识库/领域（整理分类）→ MOC 导航（链接聚合）
```

### 4. Memory ↔ Obsidian 双向同步
```
Claude Memory → sync_memory_to_obsidian.sh → 知识库/Claude Memory/
                                                     ↓
                                              Obsidian 内可浏览、链接、扩展
```

### 5. 链接自生长
```
新建笔记 → 自动关联 [[🏠 首页]]
                → 手动添加领域链接
                → Graph View 自动发现新关系
                → 诞生新的 MOC 需求
```

---

## 收件箱工作流

```
Inbox（知识库/收件箱/）
    │
    ├── 每周整理一次
    │
    ├── 可转化为知识笔记 → 知识库/[领域]/ 
    ├── 可转化为任务 → 工作日志/日报
    ├── 可关联到现有笔记 → 添加 [[链接]]
    └── 信息性暂存 → 保留在收件箱
```

---

## 自动化引擎

| 脚本 | 触发时机 | 作用 |
|------|----------|------|
| `regenerate_all.sh` | 每次生成文档 | 自动记录变更到 [[变更日志]] |
| `sync_memory_to_obsidian.sh` | 手动运行 | Claude Memory → Obsidian |
| Obsidian Templater | 新建笔记 | 自动填充格式和链接 |
| Dataview | 打开首页 | 自动汇总最近活跃/待办 |

---

## 生长指标

| 维度 | 当前状态 | 目标 |
|------|----------|------|
| 知识笔记数 | 10+ | 每周+3 |
| MOC 覆盖率 | 核心模块已覆盖 | 全模块覆盖 |
| 双向链接密度 | 中等 | 每篇笔记 ≥3 链接 |
| Memory 同步 | 已建立 | 双周同步一次 |
