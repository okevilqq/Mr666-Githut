梁君衡是链邦赋商通（原链商2.0）项目的企业宣传策划专员，负责品牌策划、文档生成、命名方案、营销策略等工作。

**用户画像：**
- 非技术背景，不熟悉技术术语
- 需求描述可能口语化、不完整、含糊
- 脾气直接，不喜欢绕弯子

**核心工作原则 —— 产品经理思维优先：**
1. 收到需求时先理解意图再动手——用户说的不一定是他真正要的
2. 口语化描述 → 翻译成清晰的功能需求 → 再执行
3. 遇到歧义时给出自己的理解并确认，而不是直接乱做或反复追问
4. 回答时先给结论再给过程，不做选项清单直接给推荐方案
5. 用业务语言沟通，不堆砌技术术语

**Why:** 用户是企宣策划专员，关注品牌/内容/业务成果，不关心技术实现细节。

**工作风格：**
- 追求高质量输出，愿意通过迭代优化方案（如品牌命名从V2到V4到赋商版→链邦赋商通）
- 重视合规性和风险控制，所有文档必须遵守合规术语映射表
- 偏好结构化、系统化的文档组织方式（10章+附录是标准结构）
- 认可分润模型、财务指标的精确性，使用精确百分比而非模糊表述

**格式偏好：**
- 中文文档使用微软雅黑字体，21pt正文
- 表格风格偏好深海蓝(#1A5276)主色调
- 品牌色偏好：链商红(#D62828)用于画册设计，深海蓝用于商务文档
- Excel工作计划表使用9色工作类型系统
- 文档章节编号偏好中文数字（第一章、一、）

**沟通习惯：**
- 决策前会收集领导会议反馈（如6/10领导会议决议）
- 偏好多方案并行评估（通常提供3套方案+6维评估）
- 重视商标注册可行性和字频分析
- 每次产出都有明确的版本号和输出目录
- 遇到模糊需求直接给判断，不反复追问

**AI工具使用：**
- 所有文档通过Node.js生成脚本产出，不手动创建
- 生成脚本运行后产出.docx/.xlsx/.html文件
- 26个活跃脚本统一使用lib/docx-helpers.js共享库

**文档输出路径规则（20260617起执行）：**
- 新文档输出到项目根目录下的 `(日期) + 文档名/` 子目录
- 格式：`YYYYMMDD 文档主题/文件名.docx`
- 示例：`20260617 逆向交易规则/链邦赋商通_逆向交易规则_V1.0.docx`
- 旧文档（20260617前）保留在原 `20260602 链商平台 技术部会议整理/` 目录
- 此规则已写入 CLAUDE.md，新建脚本时应遵守
